    multiplied.tests.analysis.conftest.cleanup_matplotlib()[source]#

    FIXTURE: Clean up matplotlib state between tests

    SCOPE: function

    PURPOSE:

    Matplotlib maintains internal state (figures, styles, etc.). This fixture ensures tests don’t interfere with each other.

    BEHAVIOR:

    • Runs BEFORE each test (setup)

    • Clears matplotlib figures

    • Resets matplotlib style

    • Runs AFTER each test (cleanup)

    • Closes any open figures

    ⚠️ autouse=True means this runs automatically, no need to request it

    WHY IT MATTERS:

    Without this: - Test 1 creates figure - Test 2 inherits figure from Test 1 - Tests interfere with each other (flaky)

    With this: - Each test starts fresh - No interference

    AI CONFIDENCE: ~95%

    Standard matplotlib testing practice.

    WHAT IT DOES:

    Before test: - Clears all figures - Resets to default style

    After test: - Closes remaining figures - Clears figure cache

    FIXTURE: Real DataFrame from algorithm truth table (small)

    SCOPE: function

    PURPOSE:

    Generates actual DataFrame using Multiplied’s truth_dataframe() function. This is real data, not synthetic, so tests are more realistic.

    GENERATION:

    1. Creates basic 4-bit algorithm

    2. Generates truth scope (1-15 range)

    3. Creates truth dataframe from algorithm

    USAGE:
    def test_with_real_data(real_truth_dataframe_small):

    mp.df_global_heatmap(“test.svg”, “Title”, real_truth_dataframe_small) # Test with actual algorithm output

    Returns:

    pd.DataFrame generated by mp.truth_dataframe()

    AI CONFIDENCE: ~85%

    This should work reliably, but algorithm changes could affect it. Takes time to generate, so use sparingly.

    SCOPE NOTE:

    Using function scope (not module) because: - Tests shouldn’t share data (isolation principle) - Generation is fast enough for function scope - Reduces coupling between tests

    WHAT TO VERIFY:

    • DataFrame structure matches expectations

    • All required columns present

    • No NaN or null values
